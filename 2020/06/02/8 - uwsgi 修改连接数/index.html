uwsgi 修改最大连接数

发布于 2020-06-02 in 并发
本文字数：313 字 | 阅读时长 ≈ 1 分钟

uwsgi

概述

用 uwsgi 启动 web 服务时，看到这么一句话：your server socket listen backlog is limited to 100 connections

如何修改连接数

最简单的方法就是启动服务时使用 -l or --listen 选项

-l|--listen <num>
       set  socket  listen queue to <n> (default 100, maximum is system
       dependent)

配置文件中用 listen=<num>

但是，当修改连接数大于 128 时，uwsgi 启动服务会失败

这是因为在 Unix scoket 和 TCP 连接监听队列上也有系统级别限制，缺省值为128，需要修改这个限制

可以通过以下命令来验证 cat /proc/sys/net/core/somaxconn

如果想突破该限制，可用以下命令：

$ echo 4096 > /proc/sys/net/core/somaxconn
或者
$ sysctl -w net.core.somaxconn=4096

或者添加 net.core.somaxconn=4096 到 /etc/sysctl.conf 使其在重启后仍然存在

如果使用 docker 或 docker-compose 呢

  1. 如何在内核中提高连接数

在 docker-compose.yml，可以这样做

uwsgi_runner:
    <<: *app-base
    command: /usr/local/bin/uwsgi --ini /app/uwsgi.ini
    # ... other settings ...
    sysctls:
        net.core.somaxconn: 1024 # set max connections to 1024 in kernel
  1. 如何在 uwsgi 中提高连接数

在 uwsgi.ini：

[uwsgi]
# ... other settings ...
listen = 1024 # set max connections to 1024 in uWSGI

或者用命令选项：

uwsgi_runner:
    <<: *app-base
    command: /usr/local/bin/uwsgi -l 1024 #other-parameters-here
